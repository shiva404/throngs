Saliency Heatmap (Eye-Tracking Prediction)

Spectral Residual algorithm predicts where the human eye is drawn. Blue = ignored, Green = noticed, Yellow/Red = immediate fixation.

Visual Perception Engine (4-Phase Pipeline)

Phase 1: DOM penalties (size, contrast, color) — Phase 2: Saliency — Phase 3: True Visibility Score + blindspot — Phase 4: Cognitive overload detection.

Frustration Engine

Calculates dynamic frustration from cognitive load heuristics: visual clutter, jargon density, action loops, familiarity decay, visual overload.

Proximity Anxiety Detection

Detects when critical/destructive buttons are too close together (below WCAG min spacing). Triggers hesitation for low risk-tolerance personas. Uses captured page elements or manual element specification.

Hesitation / Risk Aversion Engine

LLM-driven risk analysis dynamically classifies any element + page context. Falls back to regex for known keywords (pay, submit, delete...). Risk level is compared against persona's risk_tolerance to decide whether to trigger.

Distraction Engine (LLM-Driven Chaos Monkey)

Generates persona-aware, context-sensitive interruptions. The LLM invents realistic distractions matching who the persona is and what they're doing — a bakery owner gets a supplier call, a startup founder gets an investor Slack ping, a new hire gets pulled into a meeting.

A11y Tree Formatter

Formats the a11y element list into the text representation sent to the LLM. Elements below the blindspot threshold are excluded — the LLM doesn't know they exist.

F-Pattern / I-Pattern Geographic Weighting

Divides the viewport into a 3x3 grid. Each sector has a multiplier reflecting eye-tracking research on reading patterns. Desktop uses F-Pattern, mobile uses I-Pattern.
